# APPENDIX C  
SAMPLE SCREEN OUTPUTS

This appendix presents screenshots of the key user interface screens in the ENGAGIUM system.

> **Note:** Screenshots will be inserted upon completion of the system interface.

---

## C.1 Login Page

**Purpose:** Allows instructors to authenticate and access the dashboard.

**Key Elements:**
- Email input field
- Password input field (masked)
- Forgot Password link
- Login button
- Register link for new users
- Error message display area

## C.2 Dashboard (Home)

**Purpose:** Provides an overview of the instructor's classes, recent sessions, and key statistics.

**Key Elements:**
- Navigation bar with links to main sections
- Statistics cards (total classes, students, active sessions, total sessions)
- Recent sessions list with status indicators
- Quick action buttons
- Notifications panel

[Screenshot: Dashboard Home - TBD]

---

## C.3 Class Management

**Purpose:** Allows instructors to view, create, edit, and manage their classes.

**Key Elements:**
- Create New Class button
- Filter and search controls
- Class cards displaying name, section, student count, session count, schedule, and status
- View, Edit, and More actions per class

[Screenshot: Class Management Page - TBD]

---

## C.4 Session Live Feed

**Purpose:** Displays real-time participation events during an active session.

**Key Elements:**
- Session status indicator and information
- Current participant count and summary
- Participation statistics by event type
- Chronological event stream with timestamps
- End Session button

[Screenshot: Live Feed Page - TBD]

---

## C.5 Attendance Records

**Purpose:** Displays detailed attendance information for a completed session.

**Key Elements:**
- Session header with class name, date, duration, and status
- Attendance summary (present, late, absent counts)
- Tab navigation (Attendance, Participation, Summary)
- Filter and search controls
- Export CSV button
- Attendance table with student name, status, timestamps, and duration
- Pagination controls

[Screenshot: Attendance Records - TBD]

## C.6 Participation Analytics

**Purpose:** Displays aggregated participation metrics and engagement trends.

**Key Elements:**
- Date range and class filters
- Summary cards (total events, average attendance, top student, active days)
- Participation by type chart
- Top participants ranking
- Low participation alerts

## C.7 Exportable Reports

**Purpose:** Allows instructors to export attendance and participation data.

**Key Elements:**
- Report type selection (Attendance, Participation, Combined)
- Class selector
- Date range picker
- Session selection checkboxes
- Export format options (CSV, PDF)
- Preview summary
